# Security Policy

## Supported Versions

Security fixes are applied to the latest released version of `ivn`.

## Reporting A Vulnerability

Please do not open public GitHub issues for security reports.

Instead, report vulnerabilities privately through GitHub Security Advisories:

- <https://github.com/boona13/ivn/security/advisories/new>

Include:

- A clear description of the issue
- Reproduction steps or a proof of concept
- Impact assessment if known
- Any suggested mitigation

If GitHub private reporting is temporarily unavailable, contact the repository owners privately through GitHub before public disclosure.

We will aim to:

- Acknowledge receipt within 3 business days
- Confirm severity and impact as quickly as possible
- Ship a fix or mitigation before public disclosure when feasible

We will coordinate public disclosure after a fix or mitigation is available whenever possible.

Please avoid sharing exploit details publicly until a fix is available to users.
